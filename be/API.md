# Mimmis – Backend API Reference

> **Base URL:** `http://<host>/api/v1`  
> **Content-Type:** `application/json` (request & response)  
> **IDs:** Users use UUIDv7 strings. Costs and fixed costs use auto-generated `int32` serials. Categories use caller-supplied string keys (max 50 chars).

---

## Common shapes

### Error response
Returned on any non-2xx status.

```json
{
  "code": 404,
  "message": "user not found"
}
```

| Field     | Type   | Description            |
|-----------|--------|------------------------|
| `code`    | int    | HTTP status code       |
| `message` | string | Human-readable reason  |

---

## Health

### `GET /health`

Returns the liveness status of the service. Intentionally outside the versioned group.

**Response `200`**
```json
{
  "status": "ok",
  "timestamp": "2026-02-22T10:00:00Z"
}
```

## Categories `/api/v1/categories`

Classification labels that can be attached to costs and fixed costs.
The `id` is a **human-readable string key** chosen by the caller (not a UUID), with a maximum length of 50 characters.

### Model

```json
{
  "id":          "groceries",
  "description": "Grocery and food purchases"
}
```

| Field         | Type   | Writable | Notes                                         |
|---------------|--------|----------|-----------------------------------------------|
| `id`          | string | yes      | Caller-defined key, max 50 chars, unique PK  |
| `description` | string | yes      |                                               |

### Endpoints

| Method   | Path                    | Description           |
|----------|-------------------------|-----------------------|
| `GET`    | `/categories`           | List all categories   |
| `POST`   | `/categories`           | Create a category     |
| `GET`    | `/categories/:id`       | Get a category        |
| `PUT`    | `/categories/:id`       | Update a category     |
| `DELETE` | `/categories/:id`       | Delete a category     |

#### `GET /categories` → `200`
```json
[
  { "id": "groceries", "description": "Grocery and food purchases" }
]
```
Returns `[]` when no categories exist.

#### `POST /categories` → `201`
```json
{ "id": "groceries", "description": "Grocery and food purchases" }
```
`id` is mandatory and must be unique (max 50 chars).

#### `GET /categories/:id` → `200` / `404`

#### `PUT /categories/:id` → `200` / `404`
Only `description` can be updated via PUT. The `id` in the body is ignored — the URL param is authoritative.

#### `DELETE /categories/:id` → `204` / `404`
No response body. Setting a category to null on dependent costs/fixed costs is handled automatically (`ON DELETE SET NULL`).

### Model

```json
{
  "userId":  "019551b3-...",
  "name":    "Alice",
  "surname": "Rossi",
  "mail":    "alice@example.com"
}
```

| Field     | Type   | Writable | Notes                          |
|-----------|--------|----------|--------------------------------|
| `userId`  | string | no       | Set by server (UUIDv7)         |
| `name`    | string | yes      |                                |
| `surname` | string | yes      |                                |
| `mail`    | string | yes      |                                |

### Endpoints

| Method   | Path               | Description        |
|----------|--------------------|--------------------|
| `GET`    | `/users`           | List all users     |
| `POST`   | `/users`           | Create a user      |
| `GET`    | `/users/:id`       | Get a user         |
| `PUT`    | `/users/:id`       | Replace a user     |
| `DELETE` | `/users/:id`       | Delete a user      |

Returns `[]` when no users exist.

#### `GET /users/:id` → `200` / `404`
#### `PUT /users/:id` → `200` / `404`
`userId` in the body is ignored — the URL param is authoritative.

#### `DELETE /users/:id` → `204` / `404`

## Costs `/api/v1/costs`

Variable / one-off spending entries linked to a user.

### Model

```json
{
  "id":         42,
  "userId":     "019551b3-...",
  "categoryId": "groceries",
  "date":       "2026-02-22T00:00:00Z",
  "total":      1500,
  "note":       "Grocery run",
  "name":       "Supermarket",
  "refMonth":   2,
  "refYear":    2026
}
```

| Field        | Type   | Writable | Notes                                     |
|--------------|--------|----------|-------------------------------------------|
| `id`         | int32  | no       | Auto-generated by DB (SERIAL)             |
| `userId`     | string | yes      | Must reference an existing user           |
| `categoryId` | string | yes      | Optional FK to `category.id`; pass `""` for none |
| `date`       | string | yes      | RFC 3339 timestamp                        |
| `total`      | int64  | yes      | Amount in minor currency units (e.g. ¢)  |
| `note`       | string | yes      | Free-text annotation                      |
| `name`       | string | yes      | Short label for the expense               |
| `refMonth`   | int    | yes      | Reference month (1–12)                   |
| `refYear`    | int    | yes      | Reference year (e.g. 2026)               |

### Endpoints

| Method   | Path              | Description        |
|----------|-------------------|--------------------|
| `GET`    | `/costs`          | List all costs     |
| `POST`   | `/costs`          | Create a cost      |
| `GET`    | `/costs/:id`      | Get a cost         |
| `PUT`    | `/costs/:id`      | Replace a cost     |
| `DELETE` | `/costs/:id`      | Delete a cost      |

> `:id` is an **integer** (e.g. `/costs/42`). A non-integer value returns `400`.

#### `POST /costs` → `201`
Omit `id`; the server returns it in the response body.

#### `PUT /costs/:id` → `200` / `404`
`id` in the body is ignored — the URL param is authoritative.

#### `DELETE /costs/:id` → `204` / `404`
No response body.

Recurring fixed expenses linked to a user (e.g. subscriptions, rent).

### Model

```json
{
  "id":         7,
  "userId":     "019551b3-...",
  "categoryId": "subscriptions",
  "applyDay":   1,
  "expense":    49900,
  "enabled":    true,
  "note":       "Netflix"
}
```

| Field        | Type   | Writable | Notes                                       |
|--------------|--------|----------|---------------------------------------------|
| `id`         | int32  | no       | Auto-generated by DB (SERIAL)               |
| `userId`     | string | yes      | Must reference an existing user             |
| `categoryId` | string | yes      | Optional FK to `category.id`; pass `""` for none |
| `applyDay`   | int    | yes      | Day of month the expense applies (1–31)    |
| `expense`    | int64  | yes      | Amount in minor currency units              |
| `enabled`    | bool   | yes      | Whether the recurring cost is active        |
| `note`       | string | yes      | Label / description                         |

### Endpoints

| Method   | Path             | Description              |
|----------|------------------|--------------------------|
| `GET`    | `/fixed`         | List all fixed costs     |
| `POST`   | `/fixed`         | Create a fixed cost      |
| `GET`    | `/fixed/:id`     | Get a fixed cost         |
| `PUT`    | `/fixed/:id`     | Replace a fixed cost     |
| `DELETE` | `/fixed/:id`     | Delete a fixed cost      |

> `:id` is an **integer** (e.g. `/fixed/7`). A non-integer value returns `400`.

Same POST/PUT/DELETE behaviour as Costs.

## Development notes

- **Stack:** Go · Gin · PostgreSQL
- **Router:** `internal/router/router.go` — add new resource groups here.
- **Handler pattern:** each resource has its own `*Handler` struct in `internal/handler/`. Call `handler.Register(rg)` inside the router to attach routes.
- **Repository pattern:** data access lives in `internal/repository/`. Handlers never touch the DB directly.
- **ID strategies by resource:**
  - `users` → UUIDv7 string, server-generated (`internal/uid/uuid7.go`)
  - `costs` / `fixed_costs` → `SERIAL` int32, DB-generated; retrieved via `RETURNING id`
  - `category` → caller-supplied string key, max 50 chars
- **Error shape:** always use `writeError(c, status, message)` from `internal/handler/helpers.go`.
- **Integer URL params:** use `parseIntID(c, "id")` from helpers for cost/fixed-cost endpoints.
- **Nullable FK:** `category_id` is stored as `NULL` when sent as `""`. `COALESCE` in SELECT normalises it back to `""`.
- **Migrations:** `internal/db/migrate.go` — schema order matters: `category` must exist before `costs` and `fixed_costs` reference it.

### Adding a new resource (checklist)
1. Define the model struct in `internal/model/<resource>.go`.
2. Write repository methods (`GetAll`, `GetByID`, `Create`, `Update`, `Delete`) in `internal/repository/<resource>.go`.
3. Create `internal/handler/<resource>.go`, embed the repo, implement `Register(rg)`.
4. Wire it up in `internal/router/router.go` and `cmd/api/main.go`.
5. Add DB migration SQL if the schema changes.
